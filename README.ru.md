# Ностальгический снос — открытые материалы

*([English version](./README.md))*

Открытые материалы к статье **«Ностальгический снос: неизбежность устаревания памяти в нестационарных средах»** (русский — основной язык; включён английский перевод), поданной в *Journal of Physics A: Mathematical and Theoretical* (IOP).

Любая система, которая учится в меняющемся мире, должна где-то хранить выученное, а удержание памяти — по принципу Ландауэра — стоит энергии. Но мир продолжает меняться, и сохранённая память постепенно перестаёт что-либо полезно предсказывать, при этом её удержание всё равно оплачивается. Эту растущую долю бесполезной, но всё ещё оплачиваемой памяти мы называем *информационной ностальгией*, а неустранимую скорость, с которой она уводит эффективность памяти к нулю, — *сносом*. Статья доказывает, что снос **неизбежен** для медленно дрейфующих, перемешивающихся сред, каким бы маршрутом бегства система ни пыталась уйти:

```
η_v = I_pred / I_mem          (предсказательная эффективность: предсказательная информация / вся удерживаемая память)
```

- **Теорема 1** — точнее следить за миром не помогает (no-go реализуемости: адиабатический логарифм цены слежения недостижим как асимптотический наклон).
- **Теорема 2** — более частое обновление памяти оставляет неустранимый пол ностальгии, одной и той же глубины для любого класса перемешивающегося дрейфа.
- **Теорема 3** — никакой конечный полиномиальный рост ёмкости не спасает: растущая память разбавляет свою предсказательную долю, так что η_v → 0; сама ностальгия удерживается ниже единицы (у пола Теоремы 1, ν_C1 < 1), а даже экспоненциальный рост оплачивается расходящейся ценой.

Три a-priori-независимых маршрута проваливаются по одной причине — *декорреляции*: под меняющейся средой старая память неизбежно перестаёт совпадать с настоящим. Устаревание памяти — единая структурная невозможность, а не три отдельных факта. Каждая теорема несёт фальсифицируемое предсказание, подтверждённое воспроизводимыми симуляциями ниже; все результаты доказаны в линейно-гауссовом (адиабатическом) приближении, а неизбежность установлена для теоретической, эталонно-нормированной меры устаревания.

Это третья статья в серии об эффективности памяти, после [andriishin/landauer-self-modeling-oa](https://github.com/andriishin/landauer-self-modeling-oa) (статья #1, стационарная шкала) и [andriishin/landauer-nostalgia-oa](https://github.com/andriishin/landauer-nostalgia-oa) (статья #2, нестационарное расширение). **Она самодостаточна** — наследуемые определения кратко изложены в § 2; читателю предыдущие статьи не нужны.

## Воспроизведение симуляций

Каждая директория `simulations/<name>/` автономна. С Python 3.10+ и `numpy` + `matplotlib`:

```bash
cd simulations/ou_adiab
pip install -r requirements.txt
python main.py            # пишет results_summary.{txt,json}, run.log и рисунки
```

Прогоны детерминированы (фиксированные зёрна), поэтому перегенерированные `results_summary` и рисунки совпадают с закоммиченными здесь бит-в-бит. Числа из § 6 рукописи и § S4 дополнительных материалов читаются из этих файлов.

## Пересборка PDF

Готовые PDF включены для прямого чтения. Для пересборки из `.tex` нужен дистрибутив TeX с `pdflatex` и `bibtex` (TeX Live или MiKTeX). Файлы шаблона IOP (`iopart.cls`, `iopart1?.clo`, `iopams.sty`, `setstack.sty`) приложены, а bib-стиль `iopart-num` входит в TeX Live — дополнительных загрузок не требуется. Стандартный цикл для любого документа, напр. `main`:

```bash
cd paper
pdflatex main && bibtex main && pdflatex main && pdflatex main
```
