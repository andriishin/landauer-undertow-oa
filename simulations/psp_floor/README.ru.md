# `psp_floor` — пол ностальгии для пуассоновского сброса (Теорема 2, C2)

Статья #3 (`landauer-undertow-oa`) § 6 пункт 3 / supplementary § S4.3.

## Что вычисляет

Кусочно-стационарный пуассоновский (PSP) дрейф: логиты `θ(t)` кусочно
постоянны, точки сброса образуют пуассоновский поток интенсивности `μ = 1/τ_E`, и при
каждом сбросе `θ` независимо переразыгрывается («drift-with-reset»-суррогат
статьи #2 § 6.1). Перемешивание экспоненциальное: бит возраста `Δ` сохраняет предсказательную
ценность тогда и только тогда, когда на его координате не произошло ни одного сброса с момента записи.

Это **симуляция реальной траектории**. Замкнутая форма `ν(t) = e^{−μΔ}` была бы
тавтологичной; эта версия действительно **симулирует пуассоновский поток сбросов**:
каждая из `K` координат логита
несёт реализованные пошаговые часы сброса (`p_reset = 1 − e^{−μ}`).
FIFO-память на `|M| = 400` снимков-битов хранит для каждого бита метку координаты и
время записи; бит остаётся предсказательным, пока его координата **не сбросилась** с момента
записи, и становится ностальгическим в тот момент, когда это происходит. FIFO обновляет `dM`
старейших битов за шаг (доля свежих `c = τ_E/refresh_period = c_target`).

Отчёты, **считанные из реализованного процесса сбросов**:

- пол `liminf ν` для каждого `μ` (должен равняться `1 − c`, инвариантен по `μ`);
- экспоненциальная скорость декорреляции, оцениваемая по реализованной **кривой выживания**
  `Ŝ(age)` = доля сохранённых битов заданного возраста, которые ещё не видели
  сброса, фитированная `ln Ŝ ~ −rate·age` на `[0, 5 τ_E]` с bootstrap-ДИ по
  прогонам. Скорость — **эмпирический выход**, не утверждается, что она равна `μ`.

Это покрывает **экспоненциальный конец** класса перемешивания, дополнительный к
степенному концу в `fou_floor`.

## Параметры

- `k = 8`, `K = 56`, `μ ∈ {10⁻³, 3·10⁻³, 10⁻²}`, `c_target = 0.30`
- `|M| = 400`, `T = 20000`, `N_runs = 40`, `measure_every = 50`, `N_boot = 400`
- seed `20260528` (seed статьи #2 + 4), смещение на каждое `μ`

## Как запустить

```
cd simulations/psp_floor
pip install -r requirements.txt
python main.py
```

Время выполнения ≈ 28 с. Выходы: `results_summary.{txt,json}`, `run.log`,
`fig_psp_nu_vs_mu.png`, `fig_psp_approach.png`.

## Фактический результат (фиксированный seed, реализованные потоки сбросов)

| μ      | τ_E  | пол (эмпирический) | эмпирическая скорость | 95% ДИ               | номинальное μ |
|-------:|-----:|------------------:|---------------:|:---------------------|----------:|
| 1e−3   | 1000 | 0.7127            | 1.010e−3       | [9.85e−4, 1.039e−3]  | 1.000e−3  |
| 3e−3   | 333  | 0.7092            | 3.015e−3       | [2.978e−3, 3.051e−3] | 3.000e−3  |
| 1e−2   | 100  | 0.7093            | 1.002e−2       | [9.95e−3, 1.009e−2]  | 1.000e−2  |

- **Инвариантность пола: ПОДТВЕРЖДЕНА.** `floor ν = 0.709–0.713 ≈ 1 − c = 0.70`
  на 10-кратном диапазоне частоты переключений (сбросов) `μ`; разброс `0.0035`. Константа пола
  не зависит от `μ`. Поскольку это *реализованный* процесс сбросов (биты могут
  декоррелировать слегка раньше номинального горизонта), эмпирический пол лежит чуть
  выше 0.70 — подлинный выход, а не зашитое пороговое значение.
- **Экспоненциальная скорость ≈ μ: ПОДТВЕРЖДЕНА (по-настоящему).** Реализованная кривая выживания
  спадает как `e^{−rate·age}` со скоростью `1.010e−3, 3.015e−3, 1.002e−2`; каждый
  bootstrap-ДИ **накрывает номинальное `μ`**. Это реальное измерение
  временного масштаба декорреляции из симулированного потока сбросов, а не повторный фит
  формулы.

Отклонений от теории нет. Вместе с `fou_floor` (степенной конец) это
прорабатывает оба конца класса перемешивания; степенной показатель `fou_floor`
`4(1−H)` восстанавливается в дальнем хвосте на точном стационарном fOU (см. тот
README), а экспоненциальная скорость PSP `μ` восстанавливается здесь.
