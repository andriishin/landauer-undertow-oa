# `ou_adiab` — реализуема ли `B → K/2` при OU-дрейфе? (Теорема 1, C1)

Статья #3 (`landauer-undertow-oa`) § 6.1 / supplementary § S4.1.

## Что проверяется

Теорема 1 выводит, в линейно-гауссовом (калмановском) приближении, что
кумулятивная избыточная потеря идеального байесовского обучателя, следящего за
медленно дрейфующим OU-латентом, растёт как

```
L_excess(t) = A·t + (K/2)·ln(λt) + C + o(1),   K = k(k-1) = 56,  K/2 = 28.
```

Открытый вопрос (унаследованный из companion paper #2, § S8.3) состоял в том,
можно ли **восстановить численно** лог-коэффициент `B = K/2` как наклон: прежние
сканы на фиксированном окне `[10³,10⁴]` оставляли `B/(K/2) ≤ 0.43`, а сходимость —
открытой. Естественная гипотеза состоит в том, что скан не входил в режим слежения
`ε_track = I_rate·σ²/λ² ≪ 1`.

Этот редизайн проверяет эту гипотезу **точно в линейно-гауссовой постановке, которую
предполагает теорема** (так что любой провал — свойство BNT-лога, а не
неоптимального softmax-обучателя). Модель: `K` независимых скалярных OU-координат,
гауссовы наблюдения `y = θ + v` (`Var v = r`) и **оптимальный фильтр
Калмана** — который и *есть* идеальный байесовский обучатель. `r` фиксирует пошаговую
фишеровскую информацию `I_rate = 1/r`, поэтому `ε_track = σ²/(λ²r)`.

Три контролируемых эксперимента:

- **(A) Статический якорь** (`λ=0, σ=0`, диффузный приор): кумулятивная избыточная
  потеря *обязана* быть в точности `(K/2) ln(t)` (классическая MDL/BIC-избыточность).
  Валидирует измерительный конвейер и целевое значение `K/2`.
- **(B) Диагностика локального наклона** при OU-дрейфе: отслеживать `dL_excess/d ln t`
  вдоль всей траектории и локализовать, где он равен `K/2`.
- **(C) Асимптотический скан по широкому окну** по `ε_track` (с `r = 10⁻²`, так что
  окно `[3τ_conc, 0.1/σ²]` имеет отношение ≈150, не ≈10): фит `A·t + B·ln(λt) + C`,
  отчёт `B/(K/2)` с bootstrap-ДИ 95% и коллинеарностью регрессоров
  `corr(t, ln)`.

## Результат (ЧЕСТНО — сходимость НЕ достигнута)

- **(A)** восстанавливает `slope/(K/2) ≈ 0.99` (R² ≈ 0.9998): BNT-лог реален и
  конвейер корректен.
- **(B)** локальный лог-наклон равен `K/2` **только во время краткого начального
  всплеска коллапса приора** (`t` до нескольких/`λ`). После этого дрейф ограничивает
  эффективное число выборок на уровне `n_eff ~ 1/g*`, `L_excess` **выходит на плато**
  на высоте `~ (K/2) ln(1/g*)`, и наклон коллапсирует к ≈ 0.
- **(C)** в широком асимптотическом окне `B/(K/2) ≈ 0` (потеря там чисто
  линейна) и **не** растёт к 1 при `ε_track → 0`. Гипотеза «неверный режим,
  гнать `ε_track → 0`» **опровергнута**.

**Почему сходимость здесь структурно невозможна:** `τ_conc` (время
концентрации `1/g* ≫ τ_E`, отличное от масштаба забывания `τ_f^forget ~ τ_E`)
и `1/σ²` (граница адиабатической применимости) оба масштабируются как `1/σ²`, так что
окно, где живёт лог (транзиентный всплеск, `t < τ_conc`), и асимптотическое требование
`t ≫ τ_conc` не могут быть разделены в широкое лог-окно. Аналитическое тождество
`B = K/2` (предел `ε_track → 0`) реализуется лишь **транзиентно**; оно не
восстановимо как асимптотический наклон по широкому окну при подлинном OU-дрейфе,
независимо от того, насколько малым продавлен `ε_track`. Это отражено как структурное
численное ограничение C1 в § 6.1 / § S4.1.

## Параметры

- `k = 8`, `K = 56`, `λ = 10⁻²`, диффузный приор `P0 = 10⁶`
- `r = 1` для (A)/(B); `r = 10⁻²` для скана по широкому окну (C)
- скан `ε_track`: 10 логарифмически равномерных точек по `[3·10⁻¹, 3·10⁻²]` — диапазон, где
  лог-окно остаётся широким и σ-инвариантным (`t_max = 0.1/σ² < 4·10⁶` cap)
- `N_runs = 40`, bootstrap `N = 2000`, seed `20260527`

## Как запустить

```
cd simulations/ou_adiab
pip install -r requirements.txt
python main.py
```

Время выполнения ≈ 7 мин (доминируется самой глубокой точкой скана). Выходы:
`results_summary.{txt,json}`, `run.log`, `fig_ou_adiab_B_vs_eps.png`
(оконный `B/(K/2)`, плоский около 0, против опор статического якоря и всплеска),
`fig_ou_adiab_Lexcess_scan.png` (локальный лог-наклон: всплеск `K/2`, затем плато).
